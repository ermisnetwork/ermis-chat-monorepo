# Ermis Documentation Standards

When generating, updating, or maintaining the official documentation for the Ermis SDKs, you **MUST** adhere to the following conventions and structure. The documentation is built using [Docusaurus v3](https://docusaurus.io/).

## 1. Directory Structure & Workspaces
- **Location:** The root for documentation is `apps/docs/`.
- **Content Paths:** Markdown (`.md` or `.mdx`) files are specifically located within `apps/docs/docs/`.
  - Core SDK documentation resides in `apps/docs/docs/core-sdk/`.
  - React UI Kit documentation resides in `apps/docs/docs/react/`.
- **Launch Command:** To start the local Docusaurus development server to verify your changes, run `yarn docs` at the root of the monorepo (`ermis-chat-monorepo/`). This maps to `yarn workspace apps-docs start`.

## 2. Formatting & Markdown Standards
- **YAML Frontmatter:** Every new markdown file **MUST** include standard Docusaurus Frontmatter at the top of the file. 
  Example:
  ```yaml
  ---
  id: your-feature-id
  title: Your Feature Title
  sidebar_position: 1
  sidebar_label: Optional Short Label
  ---
  ```
- **File Extensions:** Default to `.md` for standard documentation. If embedding React components directly within the markdown, use `.mdx`.
- **Admonitions (Callouts):** Make use of Docusaurus standard callouts to draw attention to important details, rather than plain text.
  - `:::note` for standard information.
  - `:::tip` for optimization or best practice recommendations.
  - `:::info` for contextual details.
  - `:::caution` for potential pitfalls.
  - `:::danger` for critical breaking warnings.

## 3. Internal Linking
- **Relative Pathing:** Do NOT use absolute URLs (like `https://domain.com/docs/...`) when linking to other internal documents. You **must** use relative markdown links pointing directly to the target `.md` file. It ensures links don't break during versions or local testing.
  - *Correct:* `[See UI Hooks](../react/hooks.md)`
  - *Incorrect:* `[See UI Hooks](/docs/react/hooks)`

## 4. Code Blocks
- Provide accurate language identifiers for syntax highlighting (e.g. ` ```typescript `, ` ```tsx `).
- Focus on practical, copy-pasteable examples.
- When referencing other SDK functionality, ensure you follow the conventions defined in `ermis_sdk_standards` and `ermis_react_ui_standards`.

## 5. Sidebar Navigation
- If the project does not rely purely on `autogenerated` sidebar configurations, ensure you update `apps/docs/sidebars.ts` whenever you introduce a completely new section or root file. Always check if the file appears in the UI menu after creation.
